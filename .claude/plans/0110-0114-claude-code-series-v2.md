### 第八篇：踩坑经验与最佳实践
**文件名**：`0117-claude-code-pitfalls.md`
**标题**：`Claude Code 踩坑一个月，这些经验你一定要知道`
**id**：0117
**定位**：避坑指南，解决"怎么避免常见问题"的问题

**内容大纲**：
1. 为什么需要这篇（踩坑的价值）
2. **上下文管理踩坑**：
   - 问题：AI "变笨"
   - 原因：上下文窗口满了
   - 解决：`/compact`、`/clear`、精简 CLAUDE.md
3. **CLAUDE.md 配置踩坑**：
   - 问题：AI 不遵守规范
   - 原因：配置太长被忽略
   - 解决：控制在 200 行、使用强调词
4. **权限模式踩坑**：
   - 问题：改错了文件
   - 原因：用错了权限模式
   - 解决：理解 5 种模式、合理切换
5. **非交互模式踩坑**：
   - 问题：无限循环
   - 原因：没设 `--max-turns`
   - 解决：务必限制轮次
6. **MCP 踩坑**：
   - 问题：上下文消耗过大
   - 原因：MCP 服务器太多
   - 解决：只加必要的、定期检查
7. **Git 集成踩坑**：
   - 问题：无法创建 PR
   - 原因：没装 `gh` CLI
   - 解决：安装并配置 `gh`
8. **费用控制踩坑**：
   - 问题：API 费用超支
   - 原因：没有限制轮次
   - 解决：合理设置 `--max-turns`
9. **调试技巧**：
   - Plan Mode 先分析
   - Subagent 隔离
   - `/rewind` 回退
   - `git diff` 验证
10. **最佳实践总结**：
    - 上下文管理策略
    - 权限模式选择
    - 配置优化建议
    - 成本控制技巧
11. 我的使用习惯总结
